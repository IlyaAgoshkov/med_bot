# Запуск бота через Docker

## Требования

- Docker (версия 20.10+)
- Docker Compose (версия 1.29+)

## Быстрый старт

1. **Создайте файл `.env`** в корне проекта (скопируйте из примера ниже):
   ```env
   # Токен бота от @BotFather (обязательно)
   BOT_TOKEN=ваш_токен_бота_здесь
   
   # ID администраторов через запятую (опционально)
   # Получить свой ID можно у бота @userinfobot
   ADMIN_IDS=123456789,987654321
   ```

   **Важно:** 
   - Файл `.env` должен находиться в корне проекта (там же, где `docker-compose.yml`)
   - Не коммитьте `.env` в git (он уже в .gitignore)
   - BOT_TOKEN обязателен, без него бот не запустится

2. **Запустите бота:**
   ```bash
   docker-compose up -d
   ```

3. **Проверьте логи:**
   ```bash
   docker-compose logs -f
   ```

## Команды Docker Compose

### Запуск бота
```bash
docker-compose up -d
```

### Остановка бота
```bash
docker-compose down
```

### Просмотр логов
```bash
docker-compose logs -f
```

### Перезапуск бота
```bash
docker-compose restart
```

### Пересборка образа (после изменений в коде)
```bash
docker-compose build --no-cache
docker-compose up -d
```

### Просмотр статуса
```bash
docker-compose ps
```

## Структура данных

База данных сохраняется в директории `./data/` на хосте, что позволяет сохранять данные при перезапуске контейнера.

## Обновление бота

1. Остановите контейнер:
   ```bash
   docker-compose down
   ```

2. Обновите код (через git pull или вручную)

3. Пересоберите и запустите:
   ```bash
   docker-compose build --no-cache
   docker-compose up -d
   ```

## Отладка

### Вход в контейнер
```bash
docker-compose exec med_bot /bin/bash
```

### Просмотр базы данных
База данных находится в `./data/med_bot.db` на хосте. Вы можете использовать любой SQLite клиент для просмотра данных.

## Переменные окружения

Все переменные окружения задаются в файле `.env`:

- `BOT_TOKEN` - токен бота (обязательно)
- `ADMIN_IDS` - ID администраторов через запятую (опционально)

## Проблемы и решения

### Бот не запускается
1. Проверьте логи: `docker-compose logs`
2. Убедитесь, что токен бота указан правильно в `.env`
3. Проверьте, что порты не заняты

### Данные не сохраняются
Убедитесь, что директория `./data` существует и имеет правильные права доступа:
```bash
mkdir -p data
chmod 755 data
```

### Ошибка при сборке
Попробуйте пересобрать без кеша:
```bash
docker-compose build --no-cache
```
